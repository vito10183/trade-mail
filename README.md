# 📊 統一證券交易郵件自動化處理

自動從 Gmail 擷取統一證券的交易通知郵件，並將資料整理至 Google Sheets。

## ✨ 功能特色

- 📧 **自動解析郵件** - 處理「交易明細表」與「成交回報資料」兩種郵件格式
- 📈 **交易紀錄管理** - 擷取買賣資訊、價格、數量、手續費等完整交易資料
- 📦 **庫存明細追蹤** - 記錄持股狀態，包含現股、融資、融券等資訊
- 🔄 **自動去重** - 智慧識別重複資料，優先保留較完整的交易明細表資料
- 📋 **分類工作表** - 自動將 0050 與 006208 庫存資料分割至獨立工作表
- ⏰ **定時執行** - 支援每 12 小時自動觸發處理

## 🚀 使用方式

### 1. 設定 Google Apps Script

1. 開啟 [Google Apps Script](https://script.google.com/)
2. 建立新專案，將 `trade.js` 內容貼上
3. 綁定目標 Google Sheets 試算表

### 2. 授權權限

首次執行需授權以下權限：
- Gmail 郵件讀取與標籤管理
- Google Sheets 讀寫

### 3. 執行函數

| 函數 | 說明 |
|------|------|
| `processAllEmails()` | 處理所有未處理郵件（主要入口） |
| `createTimeTrigger()` | 建立每 12 小時自動執行的觸發器 |
| `resetAndProcessAll()` | 清空所有資料並重新處理 |
| `cleanupTradeSheet()` | 手動執行去重與格式修正 |
| `splitInventoryByStock()` | 手動分割庫存明細至個別工作表 |

## 📬 支援的郵件格式

### 來源
- **寄件者**: `service@mailagent.pscnet.com.tw`

### 郵件類型
1. **交易明細表** - 包含完整交易明細與庫存資訊
2. **統一證券成交回報資料** - 即時成交通知

## 🏷️ Gmail 標籤

處理完成的郵件會自動標記：
- `已處理-交易明細表`
- `已處理-成交回報`

## ⚙️ 設定說明

可在腳本開頭修改以下設定：

```javascript
var TRADE_SHEET_NAME = '交易明細';      // 交易明細工作表名稱
var INVENTORY_SHEET_NAME = '庫存明細';  // 庫存明細工作表名稱
var PROCESSED_LABEL_DETAIL = '已處理-交易明細表';  // 郵件標籤
var PROCESSED_LABEL_REPORT = '已處理-成交回報';    // 郵件標籤
```

## 📝 注意事項

- 股票代碼會自動補零（如 `50` → `0050`，`6208` → `006208`）
- 股票代碼欄位設定為純文字格式，避免前導零遺失
- 重複資料識別依據：交易日 + 股票代碼 + 價格 + 數量
- 若交易明細表與成交回報有重複，優先保留交易明細表資料
